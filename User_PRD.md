## 1. 목적
카카오톡 비즈 채널 링크로 진입한 시청자가 **현재 방송 중인 상품**을 곧바로 확인하고 **주문·결제까지 이탈 없이** 완료하는 모바일 특화 경험을 제공한다.

## 2. 페르소나
- **P1 · 구매자(게스트)** — 모바일, 카카오 비즈채널로 진입, **로그인 없음**, 방송 중 즉시 구매 성향, 무통장/간편결제 사용.

## 3. 기술 스택 (현행 유지)
React 18 · Vite 5 · TypeScript 5 · Tailwind CSS 3 · Zustand 5 · react-router-dom 6 · 인메모리 목데이터.

## 5. 기능 요구사항

### U1. 홈 — 방송중 상품 목록 🔧
현재 목업은 전체 카탈로그를 노출한다. 목표에 맞게 **라이브 편성 기반**으로 변경한다.

- **FR-U1.1** 현재 `live` 세션에 **편성된 상품만** 편성 순서대로 노출한다.
- **FR-U1.2 (fallback)** `live` 세션이 없으면 상단에 "다음 방송 예고"(가장 가까운 `upcoming`) 배너 + **홈 큐레이션**(백오피스 지정) 상품 노출. 큐레이션이 없으면 전체 카탈로그.
- **FR-U1.3 (실시간감)** 핀 상품(`pinnedId`)을 최상단 강조. 카드마다 변형 재고 기반 배지("N개 남음"/"품절임박"/"품절"). 방송중이면 라이브 헤더에 시청자 수·좋아요 수 노출.
- **FR-U1.4 (탐색)** 카테고리 칩 필터 + 상품명 검색. **편성 상품 집합 내**에서만 필터링.
- **FR-U1.5** 무료배송 진행바(무료배송 기준 미달 시 남은 금액 안내).
- **FR-U1.6** 헤더에 장바구니 배지(수량) 및 주문조회(U7) 진입점.

### U2. 상품상세 🔧
- **FR-U2.1** 이미지, 정가/할인가, 상세 설명/스펙 노출.
- **FR-U2.2 (옵션)** 색상·사이즈 선택. **선택 변형의 재고**에 따라 품절 옵션은 선택 불가/취소선. 재고 초과 수량 담기 방지.
- **FR-U2.3** 수량 스텝퍼(변형 재고 상한).
- **FR-U2.4 (장바구니 담기)** 선택 변형 추가, 동일 변형 수량 합산.
- **FR-U2.5 (원탭 빠른구매)** "바로구매" 시 장바구니를 거치지 않고 해당 변형만으로 U4로 진입.
- **FR-U2.6 (공유)** 카카오 공유/링크 복사 진입점(목업 UI).

### U3. 장바구니 ✅
- **FR-U3.1** 라인별(변형별) 수량 증감/삭제, 전체 비우기.
- **FR-U3.2** 상품금액·배송비·총 결제금액 요약 + 무료배송 진행바.
- **FR-U3.3** 재고 부족 변형은 경고 표시 및 수량 보정.

### U4~U5. 주문/배송정보 · 결제 (mock) ✅
- **FR-U4.1** 배송정보: 받는사람·연락처·주소/상세주소·배송 요청 메모(프리셋 선택 또는 직접입력).
- **FR-U4.2** 주문 요약(상품/수량/금액) 확인.
- **FR-U5.1** 결제수단: 카카오페이 / 토스페이 / 무통장입금. **백오피스 설정에서 비활성화된 수단은 미노출.**
- **FR-U5.2** 무통장 선택 시 입금 계좌·입금기한 안내, 입금자명 입력.
- **FR-U5.3** 결제는 **목업 처리**: 주문 생성 → 변형 재고 차감 → U6 이동.

### U6. 주문완료 ✅
- **FR-U6.1** 주문번호·결제금액·상품 수, (무통장) 입금 안내.
- **FR-U6.2** 주문조회(U7) 바로가기 + 카카오 채널 추가 유도(목업 UI).

### U7. 게스트 주문조회 🆕
- **FR-U7.1** **주문번호 + 전화번호**로 조회(로그인 없음).
- **FR-U7.2** 주문 상태·상품 내역·배송지·(배송중) 송장번호 노출.
- **FR-U7.3** 무통장 입금대기면 계좌/입금기한/남은시간 안내.

## 6. 데이터 모델 (상위 PRD §7 기준)
- **재고는 상품 귀속 · 옵션(색상×사이즈) 변형 단위**. 옵션 없는 상품은 단일 변형.
- 홈/상세의 상품·재고·가격 표시는 이 모델을 단일 소스로 사용.
- 주문 라인은 (productId, color, size) 변형 참조 → 해당 변형 재고에서 차감.

```
Product { id, code, name, category, tag, price, discountRate, desc, details, images,
          options:{colors[],sizes[]}, variants: Variant[] }
Variant { color, size, stock, lowStockThreshold }
LiveSession { id, date, dow, start, end, title, status, videoUrl, pinnedId, items:CurationItem[] }
CurationItem { productId, order, salePriceOverride? }
```
> 현행 목업의 `Product.stock` 단일 필드 → **변형별 재고**로 확장 필요.

## 7. 비즈니스 규칙 (User 관련)
- **판매가** = 정가 × (1 − 할인율).
- **배송비**: 소계 ≥ 무료배송 기준이면 0, 아니면 정액. 빈 장바구니 0.
- **재고 차감**: 주문 생성 시 변형 재고에서 즉시 차감.
- **방송중 판정**: `status === 'live'` 세션(동시 1개)의 편성이 홈 소스.

## 8. 반응형/UX 원칙
- **모바일 우선**. 현재 `MobileFrame`은 데스크톱 미리보기용 프레임 — 실서비스는 뷰포트 전체를 사용하는 반응형으로 대응.
- 방송 중 즉시 구매를 위한 짧은 결제 동선(홈→상세→주문/결제 3스텝, 원탭 구매 시 2스텝).

## 9. 비범위
- 구매자 로그인/회원, 실 결제(PG), 배송사 연동, 라이브 영상 embed(외부 시청), 쿠폰/적립.

## 10. 미해결 이슈
> 아래는 미해결로 두며, 고객 요구에 따라 변동/구현한다.
1. 원탭 빠른구매의 배송정보 재사용(로컬 저장) 여부 — 게스트라 매 주문 입력이 기본.
2. 홈 fallback 상세: 큐레이션 미설정 시 노출 우선순위(신상/베스트/전체).
3. 게스트 주문조회 실데이터 흐름 — 현 단계 목데이터, 향후 API 필요.
